Flattening Module¶

class spoonbill.flatten.TableFlattenConfig(split: bool, pretty_headers: bool = False, headers: Mapping[str, str] = <factory>, repeat: List[str] = <factory>, unnest: List[str] = <factory>, only: List[str] = <factory>, name: str = '')¶

Table specific flattening configuration

Parameters

  • split – Split child arrays to separate tables

  • pretty_headers – Use human friendly headers extracted from schema

  • headers – User edited headers to override automatically extracted

  • unnest – List of columns to output from child to parent table

  • repeat – List of columns to clone in child tables

  • only – List of columns to output

class spoonbill.flatten.FlattenOptions(selection: Mapping[str, spoonbill.flatten.TableFlattenConfig], exclude: List[str] = <factory>, count: bool = False)¶

Flattening configuration

Parameters

  • selection – List of selected tables to extract from data

  • count – Include number of rows in child table in each parent table

  • exclude – List of tables to exclude from export

class spoonbill.flatten.Flattener(options: spoonbill.flatten.FlattenOptions, tables: Mapping[str, spoonbill.spec.Table], language='en_US')¶

Data flattener

In order to export data correctly Flattener requires previously analyzed tables data. During the process flattener could add columns not based on schema analysis, such as itemsCount. In every generated row, depending on table type, flattener will always few add autogenerated columns. For root table: * rowID * id * ocid For child tables this list well be extended with parentID column.

Parameters

  • options – Flattening options

  • tables – Analyzed tables data

Spec Module¶

class spoonbill.spec.Column(title: str, type: str, id: str, hits: int = 0)¶

Column class is a data container to store column information

Parameters

  • title – Column human friendly title

  • type – Column expected type

  • id – Column path

  • hits – Count number of times column is set during analysis

class spoonbill.spec.Table¶

Table data holder

Parameters

  • name – Table name

  • path – List of paths to gather data to this table

  • total_rows – Total available rows in this table

  • parent – Parent table, None if this table is root table

  • is_root – This table is root table

  • is_combined – This table contains data collected from different paths

  • should_split – This table should be splitted

  • roll_up – This table should be separated from its parent

  • columns – Columns extracted from schema for split version of this table

  • combined_columns – Columns extracted from schema for unsplit version of this table

  • additional_columns – Columns identified in dataset but not in schema

  • arrays – Table array columns and maximum items in each array

  • titles – All human friendly columns titles extracted from schema

  • child_tables – List of possible child tables

  • types – All paths matched to this table with corresponding object type on each path

  • preview_rows – Generated preview for split version of this table

  • preview_rows_combined – Generated preview for unsplit version of this table

Stats Module¶

class spoonbill.stats.DataPreprocessor(schema: Mapping, root_tables: Mapping[str, List], combined_tables: Optional[Mapping[str, List]] = None, tables: Optional[Mapping[str, spoonbill.spec.Table]] = None, table_threshold=5, total_items=0, header_separator='/', language='en_US')¶

Data analyzer

Process provided schema and based on this data extracts information fromm iterable dataset

Parameters

  • schema – Dataset schema

  • root_tables – Paths which should become root tables

  • combined_tables – List of tables with data from different locations

  • tables – Do not parse schema and use this tables data

  • table_threshold – Maximum array length before system recommends it to separated to child table

  • total_items – Total objects processed
